Commands

Syntax:
  • < > : Required Argument
  • [ ] : Optional Argument
  • ( ) : Command Aliases

Информация

  • /help — Отправляет меню команд
    • Отправляет меню со всеми командами бота, разделёнными на категории
    Использование : /help
    Пример : /help
    Примечания : —
  • /ping — Задержка бота в данный момент
    • Отправляет задержку ответа бота (пинг), обозначаемую в милисекундах (ms)
    Использование : /ping
    Пример : /ping
    Примечания : —
  • /server — Информация о сервере
    • Отправляет всю основную информацию о сервере, которую боту удастся найти
    Использование : /server
    Пример : /server
    Примечания : Некоторые параметры могут быть None, что значит, что на сервере они не установлены
  • /user — Информация о пользователе
    • Отправляет всю основную информацию о пользователе, которую боту удастся найти
    Использование : /user <пользователь>
    Пример : /user @Clonexy2362
    Примечания : Desktop Discord status - статус онлайна на ПК версии, Web Discord status - статус онлайна в браузерной версии, Mobile Discord status - статус онлайна в мобильной версии

Модерация

  • /mute — Временно ограничить чат пользователю
    • Выдаёт пользователю наказание в виде timeout на указанное количество времени, подробнее про Timeout'ы:
    Гайд дискорда
    Использование : /mute <@Пользователь> <время> [причина]
    Примеры :
    /mute @Clonexy2362 13m
    /mute @Clonexy2362 23h Просто так
    /mute @Clonexy2362 14d Какой-то долгий мут, может сразу в бан?
    Примечания : Для использования команды нужны модераторские права: moderate_members (возможность выдавать timeout)
    Бот должен иметь как минимум аналогичные права
  • /unmute — Снять ограничение чата
    • Снимает с пользователя временное ограничение чата путём установления параметра времени ограничения на None
    Использование : /unmute <@Пользователь>
    Пример : /unmute @Clonexy2362
    Примечания : Для использования команды нужны модераторские права: moderate_members (возможность выдавать timeout)
    Бот должен иметь как минимум аналогичные права
  • /mutes — Список активных мутов
    • Отправляет список активных мутов на сервере с указанием времени мута для каждого пользователя
    Использование : /mutes
    Пример : /mutes
    Примечания : —
  • /clear — Удаляет сообщения
    • Удаляет сообщения в канале, в котором данная команда была использована, можно удалить до 1000 сообщений за раз
    Использование : /clear <Количество сообщений> [id сообщения удалить до которого] [id сообщения удалить после которого]
    Примеры :
    /clear 100
    /clear 150 1005047009486589992
    /clear 200 998843639524503644 1005427885957459979
    Примечания : Id сообщения можно получить нажав пкм по сообщению и копировать ID, если включен режим разработчика.
    Для использования команды нужны администраторские права: administrator
  • /warn — Выдать предупреждение
    • Выдаёт пользователю предупреждение с уникальным ID на вашем сервере
    Использование : /warn <@Пользователь> [причина]
    Примеры :
    /warn @Clonexy2362
    /warn @Clonexy2362 123
    /mute @Clonexy2362 Просто какая-то причина для варна, почему нет?
    Примечания : Для использования команды нужны модераторские права: manage_messages (возможность удалять сообщения)
  • /unwarn — Снять предупреждение
    • Снимает с пользователя предупреждение по уникальному ID на вашем сервере
    Использование : /unwarn <@Пользователь> <ID предупреждения>
    Пример :
    /unwarn @Clonexy2362 880055535
    Примечания : Для использования команды нужны модераторские права: manage_messages (возможность удалять сообщения)
  • /warns — Список предупреждений пользователя
    • Отправляет список предупреждений с их ID и указанными причинами которые выдавались на вашем сервере указанному в параметрах команды пользователю
    Использование : /warns <@Пользователь>
    Пример :
    /warns @Clonexy2362
    Примечания : —
  • /edit_warn — Изменить предупреждение
    • Изменяет указанное предупреждение по ID и его причину на новую, указанную в параметрах
    Использование : /edit_warn <ID предупреждения> <новая причина>
    Пример :
    /edit_warn 880055535 Новая причина предупреждения
    Примечания : Для использования команды нужны модераторские права: manage_messages (возможность удалять сообщения)

Экономика

  • /balance — Узнать баланс
    • Отправляет виртуальную карточку вашего баланса или баланса указанного в параметрах пользователя
    Использование : /balance [@Пользователь]
    Пример : /balance @Clonexy2362
    Примечания : —
  • /add_money — Добавить деньги
    • Добавляет указанному пользователю деньги на вашем сервере
    Использование : /add_money <@Пользователь> <деньги>
    Пример : /add_money @Clonexy2362 1000
    Примечания : Для использования команды нужны администраторские права: administrator
  • /remove_money — Забрать деньги
    • Забирает у указанного пользователя деньги на вашем сервере
    Использование : /remove_money <@Пользователь> <деньги>
    Пример : /remove_money @Clonexy2362 1000
    Примечания : Для использования команды нужны администраторские права: administrator
  • /reset balance — Сбросить баланс
    • Сбрасывает у указанного пользователя баланс до стандартно установленного на вашем сервере
    Использование : /reset balance <@Пользователь>
    Пример : /reset balance @Clonexy2362 1000
    Примечания : Для использования команды нужны администраторские права: administrator
  • /reset economics — Сбросить экономику
    • Сбрасывает баланс всех пользователей на вашем сервере до стандартно установленного на вашем сервере значения
    Использование : /reset economics
    Пример : /reset economics
    Примечания : Для использования команды нужны администраторские права: administrator
  • /timely — Забрать денежную награду
    • Выдаёт пользователю установленную на сервере награду при использовании данной команды
    Использование : /timely
    Пример : /timely
    Примечания : Команду можно использовать раз в 3 часа
  • /give — Передать валюту
    • Передаёт валюту с вашего баланса(снимает с него) на баланс указанного пользователя
    Использование : /give <@Пользователь> <деньги>
    Пример : /give @Clonexy2362 1000
    Примечания : —
  • /add-shop — Добавить роль в магазин
    • Добавляет в магазин ролей на вашем сервере указанную роль
    Использование : /add-shop <@Роль> <цена>
    Пример : /add-shop @роль 5000
    Примечания : Для использования команды нужны администраторские права: administrator
  • /remove-shop — Убрать роль из магазина
    • Добавляет в магазин ролей на вашем сервере указанную роль
    Использование : /remove-shop <@Роль>
    Пример : /remove-shop @роль
    Примечания : Для использования команды нужны администраторские права: administrator
  • /shop — Магазин сервера
    • Позволяет посмотреть магазин ролей на сервере и купить роли
    Использование : /shop
    Пример : /shop
    Примечания : —
  • /set currency — Установить символ валюты
    • Устанавливает новый символ валюты на вашем сервере, символом должен быть эмодзи или какой-то текстовый символ, буква, цифра, их набор
    Использование : /set currency <символ>
    Пример : /set currency 💵
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set start_balance — Установить начальный баланс
    • Устанавливает новый стандартный баланс на сервере для новых гостей или при /reset balance /reset economics командах
    Использование : /set start_balance <деньги>
    Пример : /set start_balance 300
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set timely_amount — Установить выплату /timely
    • Устанавливает новое стандартное количество денег, выдаваемое при прописывании /timely на сервере
    Использование : /set timely_amount <деньги>
    Пример : /set timely_amount 300
    Примечания : Для использования команды нужны администраторские права: administrator
  • /slots — Сыграть в слоты
    • Одна из казино игр бота - слоты, если вы проигрываете, то теряете всю ставку, если выигрываете, то ваша ставка умножается на некоторый множитель
    Использование : /slots <ставка>
    Пример : /slots 100
    Примечания : —
  • /blackjack — Сыграть в блэкджек
    • Одна из казино игр бота - блэкджек, если вы проигрываете, то теряете всю ставку, если выигрываете, то получаете бонус в виде поставленной ставки, если получается блэкджек, то x1.5 от поставленной ставки
    Подробнее про блэкджек
    Использование : /blackjack <ставка>
    Пример : /blackjack 100
    Примечания : —
  • /gamble — Сыграть в броски
    • Одна из казино игр бота - броски кубика, если вы проигрываете, то теряете случайный процент ставки, если выигрываете, то получаете случайный процент от ставки, вы и бот бросаете кубики 3 раза и получаете число от 1 до 18
    Использование : /gamble <ставка>
    Пример : /gamble 100
    Примечания : —
  • /wheel — Крутить колесо
    • Одна из казино игр бота - колесо, колесо и множители крутятся, а вы крутите стрелку и получаете один из множителей, назад вы получаете свою ставку, умноженную на полученный множитель
    Использование : /wheel <ставка>
    Пример : /wheel 100
    Примечания : —
  • /duel — Начать дуэль
    • Одна из казино игр бота -дуэль, вы стартуете дуэль со ставкой, кто-то её принимает и победитель получает ставку, а проигравший платит
    Использование : /duel <ставка>
    Пример : /duel 100
    Примечания : —
  • /income channel — Отключить/Включить доход в канале
    • Отключает или включает возможность зарабатывать валюту сервера в указанном текстовом канале сервера, с помощью параметров True/False(включеено выключено)
    Использование : /income channel <#Текстовый_Канал> <True/False>
    Пример : /income channel #флуд False
    Примечания : Для использования команды нужны администраторские права: administrator
  • /income min_max_message — Настроить доход за написание сообщений
    • Настраивает минимальный и максимальный доход, который пользователь получает за написание сообщений в неотключенных от системы доходов каналах
    Использование : /income min_max_message <минимальное_количество> <максимальное_количество>
    Пример : /income min_max_message 3 6
    Примечания : Для использования команды нужны администраторские права: administrator
  • /income min_max_voice — Настроить доход за написание сообщений
    • Настраивает минимальный и максимальный доход, который пользователь получает за нахождение в голосовом канале, внимание, расчёт валюты происходит после выхода пользователя из голосового чата, как и начисление ему времени за нахождение
    Использование : /income min_max_voice <минимальное_количество> <максимальное_количество>
    Пример : /income min_max_voice 3 6
    Примечания : Для использования команды нужны администраторские права: administrator
  • /income messages_per_income — Настроить кол-во сообщений для выдачи дохода
    • Настраивает количество сообщений необходимое пользователю написать для получения дохода за написание сообщений
    Использование : /income messages_per_income <число сообщений>
    Пример : /income messages_per_income 5
    Примечания : Для использования команды нужны администраторские права: administrator
  • /income role_add — Добавить выдачу раз в 12 часов валюты роли
    • Настраивает систему доходов раз в 12 часов для указанной роли с указанным количеством дохода
    Использование : /income role_add <@Роль> <доход>
    Пример : /income role_add @VIP 200
    Примечания : Для использования команды нужны администраторские права: administrator
  • /income role_remove — Убрать выдачу раз в 12 часов валюты роли
    • Настраивает систему доходов раз в 12 часов для указанной роли с указанным количеством дохода
    Использование : /income role_add <@Роль> <доход>
    Пример : /income role_remove @VIP
    Примечания : Для использования команды нужны администраторские права: administrator

Уровни

  • /level — Узнать уровень
    • Отправляет вашу карточку уровня или указанного в параметрах пользователя
    Использование : /level [@Пользователь]
    Пример : /level @Clonexy2362
    Примечания : Формула вычисления опыта нужного для следующего уровня: (7 * (уровень**2)) + 58
  • /add_exp — Добавить опыт
    • Добавляет указанному пользователю некоторое количество опыта на вашем сервере
    Использование : /add_exp <@Пользователь> <опыт>
    Пример : /add_exp @Clonexy2362 100
    Примечания : Для использования команды нужны администраторские права: administrator
  • /remove_exp — Забрать опыт
    • Убирает у указанного пользователя некоторое количество опыта на вашем сервере
    Использование : /remove_exp <@Пользователь> <опыт>
    Пример : /remove_exp @Clonexy2362 100
    Примечания : Для использования команды нужны администраторские права: administrator
  • /reset_level — Сбросить уровень и опыт
    • Сбрасывает указанному пользователю весь опыт до 0 и уровень до 1, то есть очищает до стартовой точки
    Использование : /reset_level <пользователь>
    Пример : /reset_level @Clonexy2362
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set level — Установить уровень
    • Устанавливает пользователю указанный в параметрах уровень
    Использование : /set level <пользователь> <уровень>
    Пример : /set level @Clonexy2362 88
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set min_max_exp — Установить промежуток опыта
    • Устанавливает на сервере минимальный и максимальный промежутки получаемого опыта за сообщения в чат, опыт будет начисляться между минимальным и максимальным значением случайным образом, значения могут быть равны, если вы не хотите рандомизации
    Использование : /set min_max_exp <минимальный опыт> <максимальный опыт>
    Пример : /set min_max_exp 20 30
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set level_up_messages — Вкл/выкл сообщения о повышении уровня
    • Включает или выключает на вашем сервере уведомления о том, что пользователь повысил свой уровень
    Использование : /set level_up_messages <turn on/turn off>
    Пример : /set level_up_messages <turn on/turn off>
    Примечания : Для использования команды нужны администраторские права: administrator

Приветствия и прощания

  • /set welcome_channel — Установить канал приветствий
    • Устанавливает канал для приветствий, в него будут отправляться сообщения - приветствия новых пользователей, посетивших сервер
    Использование : /set welcome_channel <#Текстовый_Канал>
    Пример : /set welcome_channel #welcome
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set welcome_message_type — Установить тип приветствий
    • Устанавливает тип приветствий, карточка(card) - изображение с приветствием, эмбед(embed) - настраиваемое текстовое embed сообщение
    Использование : /set welcome_message_type <card/embed>
    Пример : /set welcome_channel card
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set welcome_message_state — Включить/отключить приветствия

Тикеты

  • /setup_tickets — Установить канал Благодарности за нитро буст
    • Устанавливает в канал систему тикетов на сервере, отправит в данный канал сообщение для создания тикетов независимо от параметра, при указании параметра auto - создаст категорию для тикетов, в которую они будут добавляться и запомнит эту категорию, также создаст такую же категорию - архив, для тикетов, которые уже были закрыты, при указании параметра self - не будет создавать категорию для тикетов и архива и будет ждать, когда пользователь установит их командами самостоятельно
    Использование : /setup_tickets <auto/self>
    Пример : /setup_tickets auto
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set ticket_category — Установить категорию для тикетов
    • Устанавливает/изменяет категорию для тикетов на вашем сервере
    Использование : /set ticket_category <ID категории>
    Пример : /set ticket_category 924537076735676416
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set ticket_archive — Установить категорию - архив для тикетов
    • Устанавливает/изменяет категорию - архив для тикетов на вашем сервере
    Использование : /set ticket_archive <ID категории>
    Пример : /set ticket_category 924537076735676420
    Примечания : Для использования команды нужны администраторские права: administrator
  • /set ticket_support — Установить роль модератора тикетов
    • Устанавливает/изменяет роль, люди с данной ролью будут автоматически добавляться в тикет с возможностью отвечать в нём
    Использование : /set ticket_support <@Роль>
    Пример : /set ticket_support @
    Примечания : Для использования команды нужны администраторские права: administrator
